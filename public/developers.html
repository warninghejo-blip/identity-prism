01Overview

The Identity Prism API exposes a wallet's public reputation: its composite score, tier, earned badges, sybil-risk signals, and protocol-wide statistics. It lets any Solana app gate airdrops, governance, and access on a portable, sybil-resistant score instead of re-inventing anti-sybil from scratch.

All endpoints are REST over HTTPS. Responses are JSON, UTF-8 encoded. Addresses are base58 Solana public keys. All reputation data reflects public on-chain activity plus verified gameplay — no private or personal data is ever returned.

02Authentication

Read endpoints for public reputation data require an API key passed in the X-API-Key header. Keys are issued per project from the developer dashboard.

curl https://api.identityprism.xyz/api/v1/reputation/7xKXtg2CW87d97TXJSDpbD5jBkheTqA83TZRuJosgAsU \
  -H "X-API-Key: prism_live_xxxxxxxxxxxxxxxx"

Endpoints that act on behalf of a wallet (for example, exporting a user's own data) are wallet-based: the caller signs a short challenge message with the wallet's private key and submits the signature. The protocol verifies the signature against the public key — it never receives the key itself.

POST /api/v1/auth/verify
{
  "address": "7xKXtg2CW87d97TXJSDpbD5jBkheTqA83TZRuJosgAsU",
  "message": "Identity Prism auth · nonce=8f3a1c · ts=1716900000",
  "signature": "base58-encoded-ed25519-signature"
}

200 OK
{ "verified": true, "session": "sess_...", "expires_in": 900 }

03Endpoints

GET/api/v1/reputation/:address

Returns the composite score (0–1000), planetary tier, the five-pillar breakdown, and the wallet's earned badges — everything you need to gate a drop in one call.

GET /api/v1/reputation/7xKXtg2CW87d97TXJSDpbD5jBkheTqA83TZRuJosgAsU

200 OK
{
  "address": "7xKXtg2CW87d97TXJSDpbD5jBkheTqA83TZRuJosgAsU",
  "score": 774,
  "tier": "Saturn",
  "next_tier": { "name": "Jupiter", "at": 800, "points_to_go": 26 },
  "breakdown": {
    "onchain": 322, "sybilTrust": 230, "humanProof": 102,
    "social": 60, "engagement": 60

  "badges": [

GET/api/sybil/analysis?address=

Returns the sybil-risk assessment for a wallet: a trust score, a risk verdict, and the signals that contributed. Use this to decide whether to admit a wallet to a drop.

200 OK

GET/api/stats/global

Returns protocol-wide aggregate statistics. No address required. Useful for dashboards and explorers.

Errors — Failed requests return a standard JSON body: { "error": "not_found", "message": "..." } with the matching HTTP status (400 bad address, 401 bad key, 404 unscanned wallet, 429 rate limited).

04Rate Limits

Limits are enforced per API key on a rolling window. The current beta tiers:

TierRequests / minBurst
Free6020
Builder600100
PartnerCustomCustom

Every response includes X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers. On exhaustion the API returns 429 Too Many Requests with a Retry-After header.

05SDK

An official TypeScript/JavaScript SDK wraps these endpoints, handles wallet-signature auth, and exposes typed helpers. It is published alongside the public beta.

npm install @identity-prism/sdk

import { Prism } from "@identity-prism/sdk";

const prism = new Prism({ apiKey: process.env.PRISM_API_KEY });

const { score, tier } = await prism.score(address);
if (await prism.isHuman(address)) {
  // admit wallet to the drop
}
