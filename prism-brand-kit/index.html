Solana dApp launch

Required public pages

  • Privacy Policy
    Explain wallet/public-chain data usage, analytics, retention, and what you do not collect.
  • Terms / EULA
    Define non-custodial use, wallet responsibility, prohibited conduct, limits of liability, and jurisdiction.
  • Copyright / License
    Provide the public URL required by Solana dApp publishing flows.
  • Support / Contact
    Give users a clear resolution path and a real publisher contact surface.
  • Risk disclosure
    Cover blockchain finality, phishing, asset volatility, wallet security, and regional/compliance constraints.

Submission prep

dApp Store checklist

  • Release-ready APK or PWA wrapper
  • Version code / version name / signing key
  • Publisher website, email, support email
  • App icon, release icon, banner, screenshots
  • privacy_policy_url, license_url, copyright_url
  • Testing instructions and public website
  • Clean support/disclaimer messaging for wallet and token actions
Use dapp-store.template.yaml as the starting point for a future product-specific submission pack.

Naming system

Shared brand, separate products

  • Parent brand: Prism
    Keep the umbrella simple. Let each product use a second word that signals purpose.
  • Security app candidates
    Prism Sentinel, Prism Aegis, Prism Gate, Prism Shield.
  • Game / progression app candidates
    Prism Forge, Prism Quest, Prism Arena, Prism Rift.
  • Identity / social layer
    Identity Prism, Prism Profile, Prism Passport.
  • Shared utility / staking layer
    Prism Vault, Prism Nexus, Prism Core.

Utility layer

Recommended public wording

PRISM can be described as a shared purchasable ecosystem utility unit if that is how the products already work. If it is still off-chain or closed-loop today, say that plainly instead of pretending it is already a freely transferable public token.
  • PRISM may be used across products for cosmetics, ships, frames, boosts, subscriptions, passes, and other product unlocks.
  • Vault or staking mechanics should be described with separate rules, lockup terms, reward language, and explicit risk disclosure.
  • If PRISM later becomes a fully on-chain or publicly transferable token, publish a separate migration notice, updated terms, and jurisdiction/compliance limits before changing the message.
See PRISM token policy and Vault terms for reusable wording templates.

Isolation boundary

How to use this folder

  • Do
    Copy these pages into the future Prism product that needs them, then replace all placeholders and product-specific wording.
  • Do not
    Treat these files as final legal advice. They are a launch-ready template layer, not a signed legal review.
  • Keep separate
    App code, routes, and wallet flows should stay inside their own products. This folder is only for shared brand/legal/compliance assets.
