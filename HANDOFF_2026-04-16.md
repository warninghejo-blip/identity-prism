## Что сделано в этой сессии (30+ коммитов)

### Security & критические фиксы
- Race condition в `/api/blackhole/claim` — optimistic signature lock
- Jupiter verification в `verifySwapOperationTx`
- SKR buy — правильная проверка treasury ATA
- `resolvedCommissionRate` TDZ в BlackHole (ReferenceError)
- CelestialCard TDZ — `useState(isCapture)` до объявления const
- Arena challenges больше не висят после истечения срока
- Quest XP / achievement sync в правильные localStorage ключи per-game

### API v1/v2 + миграция старых юзеров
- `/api/v2/*` для нового APK, `/api/*` остаётся для старого (v1.0.32 dApp Store)
- 6 v1 compat handlers: game/coins, game/leaderboard, game/revives (GET+POST), prism/earn
- `ensureV2AccountState` — lazy migration при первом v2 запросе (seed XP, quests, forge)
- Welcome Back modal — показывает юзеру заработанный ранг из старой активности

### Экономика
- 4 staking lock tiers: 1w (1x), 1m (1.5x), 3m (2.5x), 6m (4x), соответствующие early penalties 10-25%
- Holder cap — исправлен: cap считается по реальным монетам (holder = 2x скорость, тот же ceiling)
- SKR discount полностью удалён (был 50%)
- Gravity achievements — описание "survive X seconds" → "pass X columns"
- Social badges синхронизированы с UI: Top Hunter (20+ scans), Quest Master (15+ quests)
- Composite: Constellation → Scans Done, Compares → Quests Done (с `questPts` = min(16, completed*1.1))

### UX
- In-app notification inbox + колокольчик с unread badge (триггеры: challenge, tournament, quest, payout)
- Leaderboard: клиент-кэш 30с + skeleton + rank/holder badges
- Shop: filter toggle "✓ Available" (по балансу, рангу, ownership)
- Vault: confirmation dialogs + toast с реальной суммой штрафа + SOL/SKR pill switcher
- Scanner: scan history с вердиктом (localStorage, до 20 записей)
- "How to Earn" → Daily Limits table с живым прогрессом из `/api/daily-limits`
- BlackHole: мобильный layout (2-col grid), 30s wallet signature timeout
- Update button — убран hasExistingId gate (DAS failure блокировал)

### Reconnect flow (тяжёлый баг)
- Root cause: `hasReachedHub.current` не сбрасывался при reconnect → 0% forever
- Fix: сброс на connect + при смене адреса + state machine priority branch
- Disconnect timeout 3s → 500ms
- isScanning = viewState === 'scanning' (не флэшит overlay при reconnect с кэшем)
- useWalletData: intermediate progress per-RPC-call (5→12→22→30→40→60→80→100), не скачком

### Карточка

- Subtle parallax (±10px yaw, ±8px pitch), убрана хаотичная дрожь звёзд
- Tier radial glow за планетой следует за поворотом

## ЧТО ОСТАЛОСЬ НЕ СДЕЛАНО (для следующих сессий)

### 🔴 Критичные security issues (из аудита GPT-5.4)

1. **`/api/prism/earn` пропускает unverified sources** — для `game_*`, `achievement`, `quest_daily/weekly/milestone`, `referral` нет server-side proof. Игрок с JWT может фармить без геймплея.
   - Файл: `server/helius-proxy.js` строки ~7255-7328
   - Нужно: server-side verification для каждого источника (как уже есть для first_mint, text_quest, scan_wallet, sybil_hunt, challenge_win)

2. **Quiz обходит NON_GAME_DAILY_EARN_CAP** — `/api/quiz/answer` не требует JWT, hardcoded cap 100 ответов/день
   - Файл: `server/helius-proxy.js` строки 9766-9796
   - Нужно: требовать JWT + включить в non-game bucket

3. **Quest sync можно подделать** — `/api/quest/sync` принимает `completed=true` от клиента без проверки условий
   - Файл: `server/helius-proxy.js` строки 9948-9999
   - Нужно: серверная верификация условий квеста (milestone/progress)

### 🟡 Средней важности

4. **DAILY_COIN_LIMIT = 300k** — покупка в 44-75x выше честного earn. Нужно обсудить с юзером: оставить или снизить.
5. **DAILY_QUIZ_CAP константа не используется** — реальный cap hardcoded отдельно (desync)
6. **Black Hole cap** не отдаётся в `/api/daily-limits` — UI хардкодит 500

### 🟢 Неподтверждённые баги (юзер тестирует)

- Карточка: кривое выделение табов (поправлено в последнем коммите — нужна проверка на Seeker)
- Bell уведомления: при пустой inbox страница должна показывать "No notifications yet"
- Social badges: UI показывает активный badge — проверить что realbadgeBonus теперь > 0

### 📝 Не приоритет

- Прод deploy (ждёт решение юзера после staging тестов)
- dApp Store submission v1.0.33

## Как продолжить

### Старт
1. `cd "C:/solana dapp"`
2. `git log --oneline -10` — посмотреть последние коммиты
3. Прочитать этот файл (`HANDOFF_2026-04-16.md`)

### Staging workflow
1. Правишь код
2. `git add -A && git commit -m "..."`
3. Если сервер: `scp server/helius-proxy.js bots:/opt/identityprism/helius-proxy-staging/ && ssh bots "fuser -k 8788/tcp; sleep 1; cd /opt/identityprism/helius-proxy-staging && nohup node helius-proxy.js > /tmp/staging-proxy.log 2>&1 &"`
4. Если фронт: `npm run build:staging && npx cap sync android && cd android && ./gradlew.bat :app:assembleStaging`
5. Установить: `adb install -r android/app/build/outputs/apk/staging/app-staging.apk`
6. `adb shell am force-stop com.identityprism.app.staging && adb shell am start -n com.identityprism.app.staging/com.identityprism.app.MainActivity`

### Тестирование
- Устройство: Solana Seeker (ID `SM02E4072805505`) — биометрия для JWT подписи, нельзя обойти удалённо
- JWT живёт 23 часа — один раз приложить палец, дальше тестировать удалённо через scrcpy
- scrcpy: `scrcpy --window-title "Identity Prism" --stay-awake` (запускать из cmd, не через фоновый процесс)
- Логи: `adb logcat -d | grep -iE "error|fail"` (WebView console не пробивается)

### Прод deploy (когда staging подтверждён)
1. `scp server/helius-proxy.js bots:/opt/identityprism/helius-proxy/`
2. `ssh bots "pm2 restart identity-prism-helius"` (или аналог)
3. `npm run build` (production mode) → Netlify auto-deploy с main branch
4. Release APK: `./gradlew :app:assembleRelease` → подписать → залить в Solana dApp Store
